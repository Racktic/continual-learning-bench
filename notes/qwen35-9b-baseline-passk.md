# Qwen3.5-9B × codebase_adaptation：stateless baseline 的 pass@k（k≤16）

> 2026-07-07。回答"这个模型在每道题上的单发成功率和采样上限是多少"。
> 所有数字由 16 遍相互独立的 stateless baseline 计算而来。

## 1. Setting（报数字前先说清测试条件）

| 项 | 值 |
|---|---|
| 模型 | /data/user_data/qixinx/Qwen3.5-9B，sglang 0.5.13.dev，**nothink 模板**（thinking 关） |
| 采样 | 未显式设置温度 → 服务端默认 temperature=1.0；**无生成长度上限** |
| 任务 | codebase_adaptation 官方 default（19 题：9 tablib + 10 tenacity），max_steps=40 |
| 口径 | stateless baseline：每题独立全新系统实例（无历史、无记忆）。reward = 1 − (steps−1)/40（解出，1 步解出=满分；代数上即 1.025 − 0.025×steps），未解出 = 0（task.py:639 的 1 − issue_regret/40，regret 以最优 1 步为基准） |
| 样本 | **16 遍独立 baseline**：1 遍来自 icl 正式组（2026-07-04T23-16-53 的 baseline）+ 15 遍来自专跑作业 9051987（×7）/ 9051988（×8），`clbench --baseline-only` 串行循环 |
| 估计量 | 无偏 pass@k = 1 − C(n−c,k)/C(n,k)，n=16，c=该题解出次数 |

数据位置：`results/codebase_adaptation/live/2026-07-07T{05-33-17,05-55-37,06-24-36,06-51-43,07-26-10,07-59-25,08-40-53,06-15-03,06-45-46,07-12-59,07-40-37,08-07-05,08-33-30,08-53-32,09-12-01}*/baseline.json` + `traces/2026-07-04T23-16-53.705279Z/baseline.json`。

## 2. 结论先行

1. **三层结构**：19 题在 16 次独立抽样下分成清晰三档——
   - **稳定层（4 题）**：606/628/540/584，单发成功率 75%+；
   - **彩票层（8 题）**：534/547/579/609/595/614/613/603，单发 12%~50%，抽 4 次以内基本能中一次；
   - **死题层（7 题）**：594/596/597/604/610/611/615，**16 连抽全灭**（单发成功率 <6%）。
2. **pass@k 曲线在 k=4 后趋平**：0.30 → 0.52 → 0.60 → 0.63。彩票层的题很快被抽中；死题层在"无迁移、单发重抽"下是硬墙。
3. **单遍 baseline 噪声很大**：均分 0.108±0.034，范围 [0.047, 0.163]。**此前所有 gain 都基于其中偏低的一次抽样（0.067）**——见 §5。
4. **死题层的边界（注意口径）**：pass@16 只证明这 7 题在*孤立单发*下成功率 <6%，**不证明知识迁移后依然做不出**。经验上：已有 ~46 个带历史/带记忆 run 里它们仅被解出 3 个 instance-run（604×1 in icl、594×2 in augment/mem0；594 的两次经轨迹归因均为运气，604 那次未归因）。且存在理论通路（610/611 可由"查 git log --all"类策略记忆解锁——opus 正是这么赢的）。故正确表述为"**当前测过的 memory 系统未翻动死题层**"，而非"死题层原理上不可由记忆解决"。死题集合与 icl 组死题几乎重合（614↔604 互换）是这一经验陈述的佐证。

## 4. 与 memory 实验的关系

- **采样上限（pass@16=0.632）远高于单发水平（0.30）**。memory 系统的价值可以框成"把 pass@k 的潜力搬进 pass@1"：模型会做的题让它每次都做对。
- 实证对上了：augment-w1 最大的赢家 tenacity-609（icl 1/5 → augment 5/5）恰是彩票层的题（单发 38%）——记忆把一次成功的解法蒸馏后，后续 run 稳定复用。
- 反面（经验陈述，见 §2.4 口径）：当前测过的 memory 系统对死题层基本零翻案。**已被证实可兑现的 memory 空间 = 彩票层（约 0.63 − 0.30 ≈ 0.33 的解出率空间）**；死题层能否由更好的策略记忆（如 git 考古类 lesson）解锁是开放问题，未被现有实验证伪。

## 5. 对已有 gain 数字的影响（重要）

此前全部 qwen 组的 gain 都以那次 0.067 的单遍 baseline 为参照——它落在 16 遍分布的低端（P25 以下）。若换成分布均值 0.108，各组"真实 gain"约需下修 0.04/题：

| 组 | 报告 gain（vs 0.067） | 校正 gain（vs 0.108） |
|---|---|---|
| augment-w1 | +0.167 | ≈ +0.126 |
| icl | +0.084 | ≈ +0.043 |
| replace-w1/w3 | +0.044 | ≈ +0.003（≈0） |
| mem0 | −0.051（vs 自框架 baseline 0.220） | 口径不同，不适用本校正 |

排名不受影响（所有组减同一常数），但**"replace 有没有正增益"这类边缘结论会翻**。官方口径就是单遍 baseline，故 viewer/对比页数字不改；解读时须带此 caveat。

## 6. thinking 模式的 pass@k（8 遍，同 setting 仅开思考）

Setting 差异（其余与 §1 完全一致）：默认模板（enable_thinking=true）+ sglang `--reasoning-parser qwen3`（`<think>` 段剥进 reasoning_content，json_schema 约束只作用于正文，思考不进历史、不占上下文预算）；无生成上限。n=8（作业 9054295）。

### thinking vs nothink 对照

| 指标 | thinking (n=8) | nothink (n=16) |
|---|---|---|
| 单遍均分 | **0.153±0.053** | 0.108±0.034 |
| pass@1 | **0.368** | 0.299 |
| pass@4 | 0.574 | 0.516 |
| pass@8 | 0.632 | 0.596 |
| 可解集合 | 12 题 | 12 题（同一批） |
| 死题 | 同样 7 道 | 7 道 |

要点：
1. **thinking 抬高单发成功率 +23%（0.30→0.37）、均分 +42%**。逐题看提升集中在彩票层：609（38%→**100%**）、579（38%→75%）、603/614（~12-19%→38%）；
2. **解出步数普遍更短**（628: 中位 14 vs 21 步、606: 15 vs 23、540: 18 vs 31）——"深思后动手更利落"，reward 双重受益（多解出 + 少走步）；
3. **抽样效率约等于翻倍**：thinking 的 pass@8 = nothink 的 pass@16 = 0.632；
4. **可解集合一寸未扩**：死题 7 道原样全灭（口径同 §2.4——这是"孤立单发"下的经验陈述）。thinking 提高的是"会做的题做对的概率"，没有解锁新题；
5. 唯一逆行：tablib-534（50%→25%），n 小暂不解读；
6. 成本：thinking 每题 token 约 3~5 倍（smoke 实测 2 题 104 万 token），单遍耗时约 30~40 分钟 vs nothink 约 25~35 分钟（4 卡 dp4、12 并发下差距不大，因为解出步数更短部分抵消了单步变慢）。

### 与有状态设置的初步交叉（详见后续 thinking 专项档案）

icl thinking 5-run（同日跑完）：均分 0.135、每 run 累计 2.56±1.04——**低于自己的 stateless baseline（0.153）**，也略低于 icl nothink（0.151）。即 thinking 的 baseline 红利在叠加 ICL 长历史后消失（n=5 未过显著性，方向存疑待查：候选解释是长上下文挤压思考质量/冷启动变慢——thinking 组前半 0.099 后半 0.167 有 nothink 没有的爬升形状）。
